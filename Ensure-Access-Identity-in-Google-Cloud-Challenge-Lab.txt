Task 1: Create a custom security role.

Execute this command on cloud shell to create yaml file for custom role:

nano role-definition.yaml

Replace the below content in the file:

title: "orca_storage_controller_109"
description: "Permissions"
stage: "ALPHA"
includedPermissions:
- storage.buckets.get
- storage.objects.get
- storage.objects.list
- storage.objects.update
- storage.objects.create





Ctrl+o
Enter
Ctrl+X



gcloud iam roles create orca_storage_controller_109 --project $DEVSHELL_PROJECT_ID \
--file role-definition.yaml

Task 4: Create and configure a new Kubernetes Engine private cluster


In the command replace the project id 
#Remember the cluster name has changed so change accordingly

gcloud container clusters create orca-cluster-366 --num-nodes 1 --master-ipv4-cidr=172.16.0.64/28 --network orca-build-vpc --subnetwork orca-build-subnet --enable-master-authorized-networks  --master-authorized-networks 192.168.10.2/32 --enable-ip-alias --enable-private-nodes --enable-private-endpoint --service-account orca-private-cluster-806-sa<PROJECT ID> --zone us-east1-b
